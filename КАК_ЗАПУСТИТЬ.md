# Easy Logistics — Как запустить проект

---

## Шаг 1 — Создать базу данных Neon (бесплатно, 5 минут)

1. Зайти на **console.neon.tech** → войти через GitHub
2. Нажать **"New Project"** → дать любое имя → Create
3. На странице проекта скопировать **Connection string** (выглядит как `postgresql://...`)
4. Это и есть `DATABASE_URL`

---

## Шаг 2 — Запуск локально

```bash
# Установить зависимости
npm install

# Создать файл с настройками
cp .env.local.example .env.local
# Открыть .env.local и вставить свой DATABASE_URL от Neon

# Запустить
npm run dev
```

Сайт: **http://localhost:3000**  
Админка: **http://localhost:3000/admin** (пароль: `admin123`)

---

## Шаг 3 — Деплой на Vercel

```bash
# Установить Vercel CLI (один раз)
npm install -g vercel

# Задеплоить
vercel
```

После деплоя зайти на **vercel.com** → твой проект → **Settings → Environment Variables** и добавить:

| Переменная | Значение |
|---|---|
| `DATABASE_URL` | строка подключения от Neon |
| `ADMIN_PASSWORD` | свой пароль |

Затем передеплоить: `vercel --prod`

---

## Где смотреть заявки

- Открыть `/admin` на сайте
- Ввести пароль из `ADMIN_PASSWORD`
- Все заявки хранятся в Neon PostgreSQL — **не теряются** при перезапуске

---

## Структура проекта

```
src/
├── app/
│   ├── page.tsx              ← Главная страница
│   ├── admin/page.tsx        ← Панель администратора (/admin)
│   └── api/
│       ├── contact/          ← POST /api/contact (сохранить заявку)
│       ├── calculator/       ← POST /api/calculator (рассчитать стоимость)
│       └── admin/submissions ← GET  /api/admin/submissions (список заявок)
├── components/               ← Все визуальные блоки сайта
└── lib/
    └── db.ts                 ← Подключение к Neon, функции БД
```
